2Turo Availability SynchronisationAnswered

Turo has discontinued their calendar-sync API, so automatic Turo availability syncing is no longer possible on any platform — this is a Turo-side change, not a Drive-247 limitation. As a workaround, you can use the Blocked Dates / Fleet Calendar tool in your portal to manually mark a vehicle unavailable for the dates it's booked on Turo; your booking site will respect those blocks and prevent double-bookings.

3Account Creation — "Invalid Passcode"Fixed

Resolved. The system was sending two competing verification codes (one from the sign-in service and one from our own email), and entering the wrong one produced the "invalid passcode" error every time. We've reconfigured it to send a single code. Account creation now works on the next attempt — this fix applies platform-wide.

4Insurance Stuck on "Being Reviewed"Fixed

The review process works — but the certificate is only saved and reviewed once the booking is completed. If a booking is abandoned before checkout, nothing is saved, which is why your test upload left no trace. We've reworded the message so it no longer implies a review is happening before the booking is placed. Once a booking is placed: the insurance is saved, automatically scanned, and you can Approve / Reject / Rescan it on the rental detail page.
